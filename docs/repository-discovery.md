## PHI-Safe Project Summary

CureRays CRMS is a radiation oncology clinical workflow prototype for CureRays treatment operations. It focuses on patient-course tracking, operational task ownership, document lifecycle visibility, chart readiness, fractionation log review, treatment delivery workflow, audit readiness, and a gradual transition from mock data to separated OPS and PHI persistence.

The project is built with Next.js App Router, React, TypeScript, Tailwind CSS, Prisma, and a two-database architecture that separates operational workflow state from protected health information boundaries.

## Safety Notes

This repository is prototype/de-identified pilot work. It must not contain real patient PHI or ePHI. Use mock, sample, synthetic, or de-identified records only until production authentication, deployment controls, audit logging, database hardening, and HIPAA security infrastructure are complete.

Do not paste real patient identifiers, medical record numbers, dates of birth, addresses, phone numbers, emails, insurance details, clinical notes tied to an identifiable person, or other PHI into prompts, issues, commits, logs, test fixtures, screenshots, or documentation.
